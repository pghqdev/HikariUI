Hikarion UI

Every component on one page, as bare semantic HTML. The theme switch restyles the page from ~20 tokens, shadows included. The density switch retunes spacing and control metrics across the whole page; body copy holds its size in both modes.

Elevation

Five rungs, --elevation-0 through --elevation-4, each meaning a material state. Shadows are mixed from the theme's accent, so they change with the theme. --shadow-sm, --shadow and --shadow-lg alias rungs 1, 2 and 4.

0 — flush

In the page. The hairline separates.

1 — raised

Attached: cards, fields, buttons.

2 — lifted

Picked up under the pointer.

3 — floating

Off the page: menus, tooltips.

4 — overlay

Above everything: dialogs, toasts.

Context-aware cards

An <article> is a query container, so its contents adapt to the column they sit in, not to the viewport. In a wide column a leading <figure> floats into a media column; in a narrow one it stacks, a [data-nav] run turns into a rail, and an action row takes the full width. Same markup in both.

Switch

A binary toggle: data-switch on a checkbox, plus role="switch" so it is announced as on/off rather than checked/unchecked.

Dropdown & action menu

Native popover: the browser owns open, light-dismiss, and focus. Rows are plain buttons and links. A destructive row is data-variant="danger", an inert row is aria-disabled, and a shortcut hint is a real <kbd>. With hikarion.js loaded the panel is an APG menu with arrow-key and typeahead navigation; without it, the rows stay a plain disclosure and no menu roles are claimed.

Button group & split button

[data-button-group] fuses adjacent <button>s into one control: one hairline per seam, outer corners only. A split button is the same group with a [popovertarget] half, so the chevron, panel and light-dismiss all come from [data-menu].

Tooltip

CSS-only [data-tooltip]: hover or focus to reveal. It waits --hk-tooltip-delay before showing and lingers after, so the pointer can travel onto the bubble without dismissing it (WCAG 2.2 1.4.13). It cannot be dismissed with Esc or revealed from the keyboard on a non-focusable element; where that matters, use popover="hint", a real element toggled by a [popovertarget] button. Browsers without popover="hint" fall back to the manual popover state, where hikarion.js supplies Esc and outside-click.

Toast

Transient feedback via Hikarion.toast(). Toasts auto-dismiss (the timer pauses while hovered or focused), stack bottom-end in the top layer above any open modal, and the stack caps at the viewport. The danger tone announces assertively; every other tone politely.

Shipping

Select & searchable input

No hook: <select>, <optgroup>, select multiple and <input list> + <datalist> carry every meaning themselves. The searchable case is a native combobox, with filtering and keyboard behaviour owned by the browser.

Date & time fields

input[type="date"], time, datetime-local: the same field chrome as every input, with the picker button repainted as the chevron <select> uses. The platform picker is the picker; Hikarion ships no calendar.

File upload previews

A plain <ul> placed after a [data-dropzone] label — not inside it, where the invisible file input would swallow the clicks. Each <li> holds an optional thumbnail and its filename; aria-live="polite" on the filename slot announces the selection.

Progress & meter

Native <progress> and <meter>; <output> shows a computed value.

Circular progress

The spinner with role="progressbar" becomes determinate — same hook, no JS. --hk-value paints the fill; CSS cannot read a number out of aria-valuenow, so the value is given twice. Keep them in sync.

Meter thresholds & range

A meter's empty track is the same hairline as <progress>, so a gauge at 0 reads as empty. The low / high / optimum regions carry the colour.

Density

Scroll progress

[data-scroll-progress] is an empty element that becomes a hairline reading-progress bar across the top of the viewport, driven by animation-timeline: scroll(root block). It is live on this page: scroll and watch the top edge. Purely presentational, so it carries aria-hidden and takes no pointer events. Where scroll() is unsupported, or under prefers-reduced-motion, it paints nothing rather than a position it does not know.

<div data-scroll-progress aria-hidden="true"></div>

Command palette

A native modal <dialog data-palette> holding a search field and a list of commands. Esc, the backdrop, the focus trap and focus return come from the browser. Rows are the same rows as the action menu. With hikarion.js the field filters as you type and ↑/↓/Enter drive the list; without it, every command is still listed and clickable.
